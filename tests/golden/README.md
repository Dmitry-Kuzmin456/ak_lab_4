# Golden tests

Каждый каталог содержит один интеграционный golden test для всей цепочки:

1. asm source транслируется в настоящий бинарный файл формата `AK4B`;
2. создается debug dump с адресами, hex-кодом и мнемониками;
3. модель процессора запускается с входным потоком;
4. проверяется вывод, ключевые строки debug dump и ключевые строки trace.

Файлы кейса:

- `source.asm` -- исходная программа.
- `input.txt` -- входной поток, если нужен.
- `expected_output.txt` -- полный ожидаемый вывод.
- `expected_debug.txt` -- обязательные строки debug dump.
- `expected_trace.txt` -- обязательные строки журнала.
- `superscalar` -- пустой marker-файл: запускать кейс с `--superscalar`.
- `limit.txt` -- необязательный лимит тактов для длинных программ.

Trace проверяется по устойчивым фрагментам, чтобы не хранить огромные журналы.
